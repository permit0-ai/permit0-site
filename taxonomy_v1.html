Risk Scoring Pipeline

Deterministic. 0–100. Computed, not labeled.

Every action is scored across 9 risk flags and 10 amplifier dimensions. No ML. No probability. Same input → same score every time.

9 Risk Flags

Multiple flags can fire simultaneously. Each carries a base weight representing its fraction of maximum risk.

10 Amplifier Dimensions

Weights sum to exactly 1.0. Raising one requires lowering another. Context shapes the final score.

How scoring works

1. Flags fire → OUTBOUND, EXPOSURE, FINANCIAL (based on action + entity context)
2. Amplifiers compute → destination:32, sensitivity:18, amount:25 ... (raw integers, context-dependent)
3. Normalize → raw / max ceiling → 0.0–1.0 per dimension
4. Categorize → data (25%), power (35%), control (40%) weighted blend
5. Session rules fire → preceded by file reads? rate spike? escalate.
6. Final score → 0–100. Deterministic. Same context = same number.
0–25: Allow
26–50: Allow + log
51–75: Approval required
76–100: Block or escalate
